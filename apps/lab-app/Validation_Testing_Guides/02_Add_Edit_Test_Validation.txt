This document outlines the validation rules and error messages for the "Add Test" and "Edit Test" 
functionality in the Lab Management System. The testing team should verify all validation rules work correctly.

================================================================================
1. FORM VALIDATION RULES & ERROR MESSAGES
================================================================================

1.1 CATEGORY FIELD
------------------
Validation Rules:
- Required field
- Minimum 2 characters
- Maximum 255 characters
- Only letters, spaces, and common punctuation allowed
- Real-time validation on input change
- Auto-converts to uppercase

Error Messages:
- "Category is required" (when empty)
- "String must contain at least 2 character(s)" (when too short)
- "String must contain at most 255 character(s)" (when too long)

1.2 TEST NAME FIELD
-------------------
Validation Rules:
- Required field
- Minimum 2 characters
- Maximum 255 characters
- Must start with a letter
- Only letters, numbers, and spaces allowed
- Real-time validation on input change
- Auto-converts to uppercase

Error Messages:
- "Test name is required" (when empty)
- "String must contain at least 2 character(s)" (when too short)
- "String must contain at most 255 character(s)" (when too long)
- "Name must start with a letter and contain only letters, numbers, and spaces" (invalid format)

1.3 PRICE FIELD
---------------
Validation Rules:
- Required field
- Must be a positive integer
- Maximum 2 decimal places allowed
- Real-time validation on input change
- No spinner arrows (disabled)

Error Messages:
- "Price must be a valid positive number" (when empty or negative)
- "Expected number, received string" (when invalid format)
- "Number must be greater than 0" (when zero or negative)
