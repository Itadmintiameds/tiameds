                    Route: http://localhost:3000/dashboard/lab

This document outlines the validation rules and error messages for the "Lab Management" 
functionality including Create New Lab operations in the Lab Management System.

================================================================================
1. FORM VALIDATION RULES & ERROR MESSAGES
================================================================================

1.1 LAB NAME FIELD
------------------
Validation Rules:
- Required field
- Minimum 2 characters
- Only alphabets, spaces, &, ., and - characters allowed
- Real-time validation on input change
- Auto-filters out numbers and special characters

Error Messages:
- "Lab name is required" (when empty)
- "Lab name must be at least 2 characters" (when too short)
- "Lab name should contain only alphabets, spaces, &, ., and -" (when contains invalid characters)

1.2 LAB TYPE FIELD

================================================================================
2. REAL-TIME INPUT FILTERING
================================================================================

2.1 ALPHABETIC FIELDS
---------------------
Fields: Lab Name, Director Name, City, State, Country
- Automatically removes numbers and special characters as user types
- Only allows letters (a-z, A-Z), spaces, and specific allowed characters
- No error messages needed (input is filtered in real-time)

2.2 ALPHANUMERIC FIELDS
-----------------------
Fields: Address, ZIP Code, Director Govt ID, License Number, Business Registration, Lab License, Tax ID, Certificate Number, Accreditation, Certification Body
- Allows alphanumeric characters and specific symbols
- No error messages needed (input is filtered in real-time)

2.3 PHONE FIELDS
----------------
Fields: Lab Phone, Director Phone
- Automatically removes invalid characters as user types
- Only allows numbers (0-9), spaces, -, +, (, )
- No error messages needed (input is filtered in real-time)

2.4 EMAIL FIELDS
----------------
Fields: Lab Email, Director Email
- Standard email format validation
- No input filtering (allows all characters for email format)

================================================================================
3. FORM SUBMISSION VALIDATION
================================================================================

3.1 REQUIRED FIELD VALIDATION
-----------------------------
All required fields are validated on form submission:
- Basic Info: Lab Name, Lab Type, Description, Address, City, State, ZIP Code, Country
- Contact Info: Lab Phone, Lab Email, Director Name, Director Email, Director Phone, Director Govt ID
- Legal Info: License Number, Business Registration, Lab License, Tax ID, Certificate Number, Accreditation, Certification Body, Data Privacy Agreement

3.2 FORMAT VALIDATION
---------------------
- Email format validation for Lab Email and Director Email
- Phone number format validation for Lab Phone and Director Phone
- Character validation for name fields (alphabetic only)
- Length validation for all text fields

3.3 CHARACTER VALIDATION
------------------------
- Name fields: Only alphabets, spaces, and specific allowed characters
- Phone fields: Only numbers, spaces, and phone-specific characters
- Address fields: Alphanumeric with address-specific characters
- Email fields: Standard email format

================================================================================
4. INLINE VALIDATION BEHAVIOR
================================================================================

4.1 VALIDATION TRIGGERS
-----------------------
- onBlur: Validates field when user leaves it (clicks away or tabs to next field)
- onChange: Clears errors when user starts typing/selecting
- onSubmit: Validates all fields before form submission

4.2 ERROR DISPLAY
-----------------
- Errors only show for fields that have been touched (interacted with)
- Red border and error text for invalid fields
- Errors clear immediately when user starts typing/selecting
- Specific, helpful error messages guide users

4.3 USER EXPERIENCE
-------------------
- Non-intrusive validation (no errors until user interaction)
- Immediate feedback when leaving fields
- Progressive error clearing as user corrects issues
- Visual indicators for field validation state
- Icons remain positioned correctly when errors appear

Fields: Lab Name, Lab Type, Description, Address, City, State, ZIP Code, Country

5.3 LEGAL INFO TAB
------------------
