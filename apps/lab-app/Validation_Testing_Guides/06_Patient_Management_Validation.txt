                    PATIENT MANAGEMENT - VALIDATION RULES
                    Route: http://localhost:3000/dashboard/patientdetails
================================================================================

This document outlines the validation rules and error messages for the "Patient Management" 
functionality including Update Patient operations in the Lab Management System.

================================================================================
1. FORM VALIDATION RULES & ERROR MESSAGES
================================================================================

1.1 FIRST NAME FIELD
--------------------
Validation Rules:
- Required field
- Minimum 2 characters
- Only alphabets and spaces allowed
- Real-time validation on input change
- Auto-filters out numbers and special characters

Error Messages:
- "First name is required" (when empty)
- "First name must be at least 2 characters long" (when too short)
- "First name should contain only alphabets and spaces" (when contains invalid characters)

1.2 LAST NAME FIELD
-------------------
Validation Rules:
- Required field
- Minimum 2 characters
- Only alphabets and spaces allowed
- Real-time validation on input change
- Auto-filters out numbers and special characters

Error Messages:

- "Last name must be at least 2 characters long" (when too short)
- "Last name should contain only alphabets and spaces" (when contains invalid characters)

1.3 EMAIL FIELD
---------------
Validation Rules:
- Required field
- Valid email format (user@domain.com)
- Real-time validation on input change

Error Messages:

- "Please enter a valid email address" (when format is invalid)

1.4 PHONE FIELD
---------------
Validation Rules:

- Exactly 10 digits

- Real-time validation on input change
- Auto-filters out non-numeric characters
- Maximum 10 characters

Error Messages:

----------------------

Error Messages:

1.11 DATE OF BIRTH FIELD
------------------------
Validation Rules:
- Required field
- Cannot be future date
- Cannot be more than 100 years ago
- Real-time validation on input change

Error Messages:
- "Date of birth is required" (when empty)
- "Date of birth cannot be in the future" (when future date)
- "Date of birth cannot be more than 100 years ago" (when too old)

================================================================================
2. REAL-TIME INPUT FILTERING
================================================================================

2.1 ALPHABETIC FIELDS
---------------------
Fields: First Name, Last Name, Gender, City, State
- Automatically removes numbers and special characters as user types
- Only allows letters (a-z, A-Z) and spaces
- No error messages needed (input is filtered in real-time)

2.2 ALPHANUMERIC FIELDS
-----------------------
Fields: Blood Group
- Automatically removes invalid special characters as user types
- Only allows letters (a-z, A-Z), spaces, + and -
- No error messages needed (input is filtered in real-time)

2.3 NUMERIC FIELDS
------------------
Fields: Phone (10 digits), ZIP Code (6 digits)
- Automatically removes non-numeric characters as user types
- Only allows numbers (0-9)
- Maximum length enforced (10 for phone, 6 for ZIP)
- No error messages needed (input is filtered in real-time)

2.4 ADDRESS FIELDS
------------------
Fields: Address
- Automatically removes invalid special characters as user types
- Allows letters, numbers, spaces, and common address characters (.,#-)
- No error messages needed (input is filtered in real-time)

================================================================================
3. FORM SUBMISSION VALIDATION
================================================================================

3.1 REQUIRED FIELD VALIDATION
-----------------------------
All required fields are validated on form submission:
- First Name, Last Name, Email, Phone, Gender, Address, City, State, ZIP, Blood Group, Date of Birth

3.2 FORMAT VALIDATION
---------------------
- Email format validation
- Phone number length validation (exactly 10 digits)
- ZIP code length validation (exactly 6 digits)
- Date of birth range validation (not future, not more than 100 years ago)

3.3 CHARACTER VALIDATION
------------------------
- Name fields: Only alphabets and spaces
- Phone/ZIP: Only numeric characters
- Address: Alphanumeric with common address characters
- Blood Group: Letters, spaces, + and - only

================================================================================
4. INLINE VALIDATION BEHAVIOR
================================================================================

4.1 VALIDATION TRIGGERS
-----------------------
- onBlur: Validates field when user leaves it (clicks away or tabs to next field)
- onChange: Clears errors when user starts typing
- onSubmit: Validates all fields before form submission

4.2 ERROR DISPLAY
-----------------
- Errors only show for fields that have been touched (interacted with)
- Red border and error text for invalid fields
- Errors clear immediately when user starts typing
- Specific, helpful error messages guide users

4.3 USER EXPERIENCE
-------------------
- Non-intrusive validation (no errors until user interaction)
- Immediate feedback when leaving fields
- Progressive error clearing as user corrects issues
- Visual indicators for field validation state
